# 機器人測試平台公開進度報告 v1.0

更新日期：2026-07-15  
文件用途：公開說明目前開發進度與下一階段方向。

> 本文件已排除帳號、密碼、內網位址、裝置序號、私人檔案路徑與未公開連線資訊。

## 專案概述

本專案正在建立一套模組化 ROS 2 機器人測試平台。系統將主控、語音、視覺、手部、腰部與底盤功能拆分為多台運算主機，並以微控制器負責接近硬體的馬達、伺服與感測器工作。

目前重點是先完成通信、主機部署與安全驗證，再逐步接入實體元件與整機功能。

## 已完成項目

### ROS 2 分散式基礎

- 六台 ROS 2 主機的基礎部署與互聯驗證已完成。
- 主機角色已分為主控、語音、視覺／頭部、手部、腰部與底盤模組。
- 系統採用 ROS 2 Jazzy 與 CycloneDDS 的分散式通信架構。
- 各主機的工作區、啟動方式與服務部署基準已建立。

### MCU 與主機通信

- 五片 Raspberry Pi Pico 2 W 已完成基本身分識別與 USB CDC 通信驗證。
- 正式 MCU 韌體已採用 C / Pico SDK 方向，並完成基本封包、校驗與重複命令保護測試。
- 底盤、腰部與手部的 ROS 2 MCU bridge 骨架已建立並完成初步測試。

### 語音輸出與輸入

- USB 耳麥的播放與錄音功能已完成實機驗證。
- 已完成本機中文 TTS 比較；目前採用輕量本機神經語音方案作為後續整合方向。
- 本機輕量中文 ASR 已完成可行性測試；速度可達需求，但實際耳麥中文辨識準確率尚未達正式採用標準。

### 硬體規劃

- 底盤四輪馬達、編碼器、雙馬達驅動器與避障感測器的通信／GPIO 規劃已建立。
- 腰部與底盤的多顆距離感測器，已規劃透過 I2C 多工器管理。
- 手部與腰部伺服控制採用 MCU 下層控制，避免由 ROS 主機直接承擔即時 PWM 工作。

## 進行中項目

| 項目 | 目前狀態 | 說明 |
|---|---|---|
| USB CDC 長時間穩定性 | 持續驗證 | 部分測試曾出現間歇停滯，尚未將 MCU bridge 接入正式動作控制。 |
| 實體底層元件整合 | 等待元件與實機測試 | 馬達、伺服、距離感測器與供電配置將依實測結果校正。 |
| 語音辨識 | 方案評估中 | 正在比較「交由較高效能主機按需辨識」與「外接離線 ASR 模組」兩條方向。 |
| AI agent 整合 | 前期規劃 | 將在硬體與語音流程穩定後，評估資源配置與任務協調方式。 |

## 已確認的設計原則

- 安全優先：未完成穩定性驗證的通信橋接，不進入實際馬達／伺服控制。
- 模組化：主機、MCU、感測器與上層任務邏輯彼此分層，避免單一模組故障擴大。
- 低資源設備負責 I/O：資源有限的主機優先處理感測、音訊與硬體橋接；較重的推論工作需先通過資源驗證。
- 通信規格先行：ROS 2 介面、MCU 封包與命名原則均以版本化文件管理。
- 所有實機結果以測試紀錄為準：規劃不等於已通過驗收。

## 下一階段

1. 持續排查 USB CDC 長時間停滯的根因，驗證供電、USB Hub 與資料線影響。
2. 在元件到位後，完成馬達、伺服、編碼器與距離感測器的分項測試。
3. 決定正式 ASR 架構：由較高效能主機按需辨識，或採用外接離線語音辨識模組。
4. 完成語音、任務協調與安全控制之間的 ROS 2 流程整合。
5. 在各分項驗證完成後，進行完整底盤與整機測試。

## 目前結論

系統已完成「主機互聯、MCU 基本通信、語音 I/O 與主要硬體架構」的前期基礎工作。現階段尚屬測試與驗證階段，尚未宣告具備完整自主運動或正式現場運行能力。
